# 贡献指南

感谢你对 Naval Mind 技能的兴趣！以下是贡献方式。

## 如何贡献

### 1. 补充 Naval 原文

如果你找到了未被收录的 Naval 内容：
- 博客文章
- 播客转录
- Twitter/X 线程
- 访谈视频

**步骤：**
1. Fork 本仓库
2. 添加原文到 `adk/references/naval-knowledge-base.md`
3. 更新来源索引
4. 提交 PR

### 2. 改进思维模型

如果你发现更好的思维模型应用方式：

**步骤：**
1. Fork 本仓库
2. 修改 `adk/references/naval-thinking-models.md`
3. 添加使用示例
4. 提交 PR

### 3. 添加使用示例

如果你用 Naval Mind 解决了实际问题，欢迎分享：

**步骤：**
1. 在 `examples/` 目录创建新文件
2. 格式：问题 → Naval 回答 → 结果
3. 提交 PR

### 4. 修复 Bug

如果你发现：
- 技能无法触发
- 回答不符合 Naval 风格
- 知识库错误

**步骤：**
1. 提交 Issue 描述问题
2. 或提交 PR 修复

## 提交规范

### PR 标题格式
```
[type]: description

Examples:
- [content]: Add 3 new Naval blog posts from 2025
- [docs]: Improve wealth section in thinking models
- [example]: Add startup decision case study
- [fix]: Correct trigger keywords in SKILL.md
```

### 代码风格

- Markdown 文件使用 LF 换行
- 引用 Naval 原文时注明来源
- 保持高信噪比，避免废话

## 内容标准

### 接受的贡献

✅ 真实的 Naval 原文引用
✅ 改进的思维模型应用
✅ 实用的使用示例
✅ 文档改进
✅ Bug 修复

### 不接受的贡献

❌ 非 Naval 的观点（即使相关）
❌ 过度解释或自助书风格
❌ 个人建议（而非 Naval 框架）
❌ 低信噪比的内容

## 开发流程

1. **Fork** 本仓库
2. **创建分支**：`git checkout -b feature/your-feature`
3. **修改内容**：按照上述规范
4. **测试**：在 Claude Code 或 OpenClaw 中验证
5. **提交**：`git commit -m "[type]: description"`
6. **推送**：`git push origin feature/your-feature`
7. **提交 PR**：描述你的改动

## 报告问题

提交 Issue 时请注明：
- 问题描述
- 复现步骤
- 期望行为
- 实际行为
- 截图（如有）
